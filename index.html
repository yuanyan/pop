The pop plugin toggles your hidden content on demand, via data attributes or JavaScript. It also adds .pop-open to the <html> to override default scrolling behavior and generates a .pop-backdrop to provide a click area for dismissing shown pops when clicking outside the pop.

Via data attributes

Activate a pop without writing JavaScript. Set data-toggle="pop" on a controller element, like a button, along with a data-target="#foo" or href="#foo" to target a specific pop to toggle.

<button type="button" data-toggle="pop" data-target="#myPop">Launch pop</button>

Via JavaScript

Call a pop with id myPop with a single line of JavaScript:

$('#myPop').pop(options)

Options

Options can be passed via data attributes or JavaScript. For data attributes, append the option name to data-, as in data-backdrop="".

Name type default description
backdrop boolean or the string 'static' true Includes a pop-backdrop element. Alternatively, specify static for a backdrop which doesn't close the pop on click.
keyboard boolean true Closes the pop when escape key is pressed
show boolean true Shows the pop when initialized.
remote path false

If a remote URL is provided, content will be loaded via jQuery's load method and injected into the root of the pop element. If you're using the data-api, you may alternatively use the href attribute to specify the remote source. An example of this is shown below:

<a data-toggle="pop" href="remote.html"
                        data-target="#pop">Click
                        me</a>

Methods

.pop(options)

Activates your content as a pop. Accepts an optional options object.

$('#myPop').pop({
        keyboard: false
        })

.pop('toggle')

Manually toggles a pop. Returns to the caller before the pop has actually been shown or hidden (i.e. before the shown.pop or hidden.pop event occurs).

$('#myPop').pop('toggle')

.pop('show')

Manually opens a pop. Returns to the caller before the pop has actually been shown (i.e. before the shown.pop event occurs).

$('#myPop').pop('show')

.pop('hide')

Manually hides a pop. Returns to the caller before the pop has actually been hidden (i.e. before the hidden.pop event occurs).

$('#myPop').pop('hide')

Events

Bootstrap's pop class exposes a few events for hooking into pop functionality.

Event Type Description
show.pop This event fires immediately when the show instance method is called. If caused by a click, the clicked element is available as the relatedTarget property of the event.
shown.pop This event is fired when the pop has been made visible to the user (will wait for CSS transitions to complete). If caused by a click, the clicked element is available as the relatedTarget property of the event.
hide.pop This event is fired immediately when the hide instance method has been called.
hidden.pop This event is fired when the pop has finished being hidden from the user (will wait for CSS transitions to complete). 
$('#myPop').on('hidden.pop', function (e) {
        // do something...
        })
